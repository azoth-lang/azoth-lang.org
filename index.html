Programming Language for Professional Programmers

Robust programming at scale

Enabling powerful libraries

Ideal Uses

  • Concurrent and Parallel Programing: structured concurrency and reference capabilities enable worry-free concurrency and parallelism.
  • Serverless Development: no startup time and multiple entry points
  • Correct and Robust Programs: design by contract, error handling etc.

Features

  • Reference Capabilities: also no global mutable state
  • Flexible Static Typing with Local Type Inference
  • Write Once, Compile to Anywhere
  • Structured Concurrency
  • Instant On: no startup overhead, almost immediately executing entry point function
  • Multiple Entry Points: enable serverless, supported by dead code elimination so each entry point gives compact executable.
  • guaranteed optimizations
  • asynchronous everywhere
  • Implicit Interfaces
  • Compile-Time Code Execution: user literals (can parse),
  • Optional Exception Specifications
  • Design by Contract
  • Null Safety: Fix the billion dollar mistake. All references non-null by default with powerful null handling and null coalescing
  • Paradigm Fusion: enables all different paradigms in a way that fits together. Fuses OO, Functional, Structured Concurrency, Lang. Oriented, Procedural, into a cohesive whole. Enable DDD, first class functions, default to immutable, side-effects visible
  • Language-Oriented Programming
  • term? No Special features in libraries, anything the language can do, a library can do. Not primitive types, operators that only apply to built in and can't be overloaded. No literals for built in types.
  • Package Management: not just about the language, it is about the package eco-system. Packages are versioned with an extension of semantic versioning but that can be disabled
  • Unsafe Code and C Interop: unsafe is there as an escape hatch but is restricted so that libraries can be trusted to be free from bugs due to unnecessary unsafe code.
  • Compact Executables: minimal runtime - GC and green-thread manager. Dead code elimination. Apps can be single executables or for multiple executable can be shipped with static shared libraries.

Other Uses

  • Web Development
  • Desktop Development
  • Full-stack Development: once web assembly supports GC can develop in Azoth on browser too. Concurrency features
  • Embedded

Powerful Libraries

  • Units of Measure and Quantities: underlying type and units optional generic parameters. Example Length is Length[float64, m]. But Length[int, year] or Length[yds] and Length[int, yd]. Can mix these types. If multiply Length[yd] * Length[m] the result is Length[yd*m]. Need to do a conversion to Length[m^2]
  • Structured Data: VSON, JSON, YAML, BSON, structure with different parsers
  • Embedded Queries: SQL like query as DSL
